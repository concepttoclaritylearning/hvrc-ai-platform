# HVRC_DATABASE_SCHEMA.md
## HVRC.AI Database & Storage Schema Specification v1.0

---

## 1. Zero-Server Database Architecture Overview
HVRC.AI adheres to a strict **Zero-Server Database Bloat** architecture. User chat conversations, code files, project workspaces, and API keys are **never** stored in central server SQL databases.
- **Authentication**: Managed via Supabase Auth (`@supabase/supabase-js`) for user identity.
- **Workspace Data**: Persisted in the user's personal **Google Drive** under `/HVRC.AI Workspace/`.
- **Client Cache**: Managed via browser `localStorage` (`hvrc_ai_...`).

---

## 2. Table & Entity Specifications

### 2.1 `Users` (Supabase Auth `auth.users`)
- **Purpose**: Authenticates user accounts and OAuth identity.
- **Primary Key**: `id` (uuid)
- **Fields**:
  - `id`: `UUID` NOT NULL
  - `email`: `VARCHAR(255)` UNIQUE NOT NULL
  - `raw_user_meta_data`: `JSONB` (Stores avatar URL, full name from Google OAuth)
  - `created_at`: `TIMESTAMPTZ`
  - `updated_at`: `TIMESTAMPTZ`
- **Access Rules**: Managed by Supabase Auth service role.

### 2.2 `Projects` (Google Drive / JSON Schema)
- **Purpose**: Defines an isolated project workspace.
- **Primary Key**: `slug` (string)
- **Fields**:
  - `slug`: `STRING` NOT NULL (e.g. `hvrc-web-app`)
  - `name`: `STRING` NOT NULL
  - `description`: `TEXT`
  - `createdAt`: `TIMESTAMP_ISO`
  - `updatedAt`: `TIMESTAMP_ISO`
  - `pinned`: `BOOLEAN` DEFAULT `false`
  - `status`: `ENUM('active', 'archived')`
- **Storage Location**: `Google Drive/HVRC.AI Workspace/projects/[slug]/workspace.json`

### 2.3 `Workspaces` (Client State / JSON Schema)
- **Purpose**: Stores active file explorer tree and editor tab state.
- **Primary Key**: `id` (string)
- **Fields**:
  - `id`: `STRING` NOT NULL
  - `projectSlug`: `STRING` FK -> `Projects.slug`
  - `activeFilePath`: `STRING` (e.g. `src/App.jsx`)
  - `openTabs`: `ARRAY[STRING]`
  - `previewUrl`: `STRING` (e.g. `http://localhost:5173`)

### 2.4 `Chats` & `Threads` (JSON Schema)
- **Purpose**: Stores conversation threads within a project.
- **Primary Key**: `id` (string)
- **Fields**:
  - `id`: `STRING` NOT NULL (e.g. `t-1`)
  - `projectSlug`: `STRING` FK -> `Projects.slug`
  - `title`: `STRING` NOT NULL
  - `date`: `STRING`
  - `pinned`: `BOOLEAN`
  - `messages`: `ARRAY[MessageObject]`

### 2.5 `Messages` (JSON Schema)
- **Purpose**: Represents an individual user or assistant chat message.
- **Primary Key**: `id` (string)
- **Fields**:
  - `id`: `STRING` NOT NULL
  - `threadId`: `STRING` FK -> `Chats.id`
  - `role`: `ENUM('user', 'assistant', 'system')`
  - `text`: `TEXT` NOT NULL
  - `reasoning`: `TEXT` (Optional reasoning steps)
  - `artifact`: `ArtifactObject` (Optional generated code artifact)
  - `createdAt`: `TIMESTAMP_ISO`

### 2.6 `Artifacts` (JSON Schema)
- **Purpose**: Interactive code preview artifacts generated by AI.
- **Fields**:
  - `title`: `STRING`
  - `type`: `ENUM('html', 'react', 'mermaid', 'markdown', 'csv')`
  - `content`: `TEXT` NOT NULL

### 2.7 `Knowledge` & `Collections` (JSON Schema)
- **Purpose**: Stores RAG vector embedding collections for document search.
- **Fields**:
  - `id`: `STRING` NOT NULL
  - `name`: `STRING` NOT NULL
  - `documentCount`: `INTEGER`
  - `vectors`: `ARRAY[Float32Array]`

### 2.8 `Files` & `Folders` (JSON Schema)
- **Purpose**: Code workspace files and folders.
- **Fields**:
  - `path`: `STRING` NOT NULL (e.g. `src/components/Navbar.jsx`)
  - `name`: `STRING` NOT NULL
  - `content`: `TEXT`
  - `type`: `ENUM('file', 'folder')`
  - `size`: `INTEGER`

### 2.9 `Providers` & `API Keys` (LocalStorage Schema)
- **Purpose**: Manages user-provided BYOK API keys for serverless models.
- **LocalStorage Key**: `hvrc_api_keys`
- **Fields**:
  - `openrouter`: `STRING` (sk-or-v1-...)
  - `groq`: `STRING` (gsk_...)
  - `nvidia`: `STRING` (nvapi-...)

### 2.10 `Models` (State Schema)
- **Purpose**: Represents an active or auto-fetched serverless model.
- **LocalStorage Key**: `hvrc_active_model`
- **Fields**:
  - `id`: `STRING` NOT NULL (e.g. `meta-llama/llama-3.3-70b-instruct:free`)
  - `name`: `STRING` NOT NULL
  - `provider`: `STRING` NOT NULL (OpenRouter, Groq, NVIDIA NIM)
  - `context`: `STRING` (e.g. `128k`)

---

## 3. Data Ownership & Storage Integrity
- **Personal Drive Sovereignty**: The user owns 100% of their data.
- **No Third-Party Access**: HVRC.AI cannot access, index, or sell private chat logs or code.

---

## 4. LocalStorage Schema Reference
| Key | Purpose | Type |
| :--- | :--- | :--- |
| `hvrc_active_provider` | Active serverless provider ID (`openrouter`, `groq`, `nvidia`) | `string` |
| `hvrc_active_model` | Selected model object | `JSON string` |
| `hvrc_api_keys` | Encrypted map of BYOK API keys | `JSON string` |
| `hvrc_ai_user` | Auth user info | `string` |
| `hvrc_ai_authToken` | Session token | `string` |
